AMAZON AURORA

¿QUE ES AMAZON AURORA?


Amazon Aurora (Aurora) es un motor de base de datos relacional totalmente administrado que es compatible con MySQL y PostgreSQL. Ya sabe cómo MySQL y PostgreSQL combinan la velocidad y la confiabilidad de las bases de datos comerciales de alto nivel con la simplicidad y la rentabilidad de las bases de datos de código abierto.


El código, las herramientas y las aplicaciones que usa hoy con sus bases de datos MySQL y PostgreSQL existentes se pueden usar con Aurora. Con algunas cargas de trabajo, Aurora puede ofrecer hasta cinco veces el rendimiento de MySQL y hasta tres veces el rendimiento de PostgreSQL sin necesidad de realizar cambios en la mayoría de sus aplicaciones existentes.

Clústeres de base de datos de Amazon Aurora

Un clúster de base de datos de Amazon Aurora consta de una o más instancias de base de datos y un volumen de clúster que administra los datos de esas instancias de base de datos. Un volumen de clúster de Aurora es un volumen de almacenamiento de base de datos virtual que abarca varias zonas de disponibilidad, y cada zona de disponibilidad tiene una copia de los datos del clúster de base de datos.

Dos tipos de instancias de base de datos conforman un clúster de base de datos de Aurora:


Instancia de base de datos principal:

admite operaciones de lectura y escritura y realiza todas las modificaciones de datos en el volumen del clúster. Cada clúster de base de datos de Aurora tiene una instancia de base de datos principal.

Réplica de Aurora:
se conecta al mismo volumen de almacenamiento que la instancia de base de datos principal y solo admite operaciones de lectura. Cada clúster de base de datos de Aurora puede tener hasta 15 réplicas de Aurora además de la instancia de base de datos principal.

Versiones de Amazon Aurora

Amazon Aurora reutiliza el código y mantiene la compatibilidad con los motores de base de datos MySQL y PostgreSQL subyacentes. Sin embargo, Aurora tiene sus propios números de versión, ciclo de lanzamiento, línea de tiempo para el desuso de la versión, etc. La siguiente sección explica los puntos comunes y las diferencias. Esta información puede ayudarlo a decidir qué versión elegir y cómo verificar qué características y correcciones están disponibles en cada versión. También puede ayudarlo a decidir con qué frecuencia actualizar y cómo planificar su proceso de actualización.

Regiones y zonas de disponibilidad

Los recursos de computación en la nube de Amazon están alojados en múltiples ubicaciones en todo el mundo. Estas ubicaciones se componen de regiones y zonas de disponibilidad de AWS. Cada región de AWS es un área geográfica separada. Cada región de AWS tiene varias ubicaciones aisladas conocidas como zonas de disponibilidad.
